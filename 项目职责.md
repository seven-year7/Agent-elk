# 项目职责（AGENT-ELK，可直接用于简历/述职）

## 项目名称
AGENT-ELK：面向运维诊断的智能 Agent 平台

## 技术栈
Python、FastAPI、Elasticsearch、Redis、RAG、MCP、Docker

## 项目介绍
围绕日志排障与故障分析场景，构建了一个可落地的智能运维 Agent。系统通过大模型 `tool-calling` 自动选择日志查询与知识库检索工具，完成“日志事实采集 -> 知识证据补充 -> 诊断建议输出”的闭环，降低人工排障成本并提升问题定位效率。

## 核心职责
- 负责 Agent 主链路设计与落地，构建多轮编排机制，支持模型间自主决策调用。  
- 负责 MCP 服务化建设，基于 FastAPI 统一封装日志与知识检索能力，实现 Agent 到检索层的标准化接口，降低业务代码与 ES 细节耦合。  
- 设计并实现 DSL 安全执行机制，落地 allowlist/blocklist、字段类型校验与错误回喂重试，提升查询稳定性与安全性。  
- 搭建知识库混合检索链路，结合 BM25 + 向量召回，并采用 RRF 融合 + MMR 重排策略，平衡相关性与多样性，提升诊断证据质量。  
- 完成短期记忆能力建设，基于 Redis Hash 存储会话上下文，支持阈值压缩、硬上限保护与跨天归档，保障长对话场景的稳定性与可控性。
