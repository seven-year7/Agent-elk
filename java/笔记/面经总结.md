### 2.垃圾回收你了解哪些,着重介绍一下CMS收集器

答：主要分为新生代，永久代，老年代

​	新生代（主要存储大多数新创建的短生命周期对象）

​	永久代（现在已经被元空间替代，主要存储类信息）

​	老年代（存储时间较长的对象，大对象也会直接进入老年代）

​      常见的垃圾回收算法为：标记清除算法（效率问题，会产生内存碎片），标记整理算法，复制算法（内存利用率低），分代收集算法（新生代：复制算法，老年代：标记清除算法）

​	为什么老年代一般采用标记清除算法，新生代采用标记复制算法

​	答：一般新生代存储时间是比较短的，而且所占用内存空间比较小，所以可以采用标记复制算法，只需要付出少量的对象的复制成本就可以完成本次垃圾收集；

​      垃圾收集器：新生代收集器：serial收集器，parNew收集器，Parallel scavenge收集器

​                             老年代收集器：serial old收集器，parallel收集器，cms收集器

​      CMS收集器是一种获取最短回收停顿时间为目标的收集器，它第一次实现了让垃圾收集线程与用户线程同时工作。分为初始标记，并发标记，重新标记和并发清除。(缺点：对CPU资源敏感，采用标记清除算法会有大量空间碎片产生，无法处理浮动垃圾)

### 11.CMS和G1垃圾回收器，为什么现在生产环境主要用G1而不用CMS了？

答：G1收集器是一款面向服务器的垃圾收集器，以极高的概率满足GC停顿时间要求的同时，还具备高吞吐量的特征，它具备以下特点：并行与并发，G1可以充分使用GPU，多核环境下的硬件优势，仍然可以使用并发的方式让JAVA程序继续执行。分代收集，空间整合，G1从整体来看使用的是标记整理算法，相较于CMS收集器，能建立可预测的停顿时间模型，

具体步骤为：初始标记，并发标记，最终标记，筛选回收

CMS缺点：内存碎片问题，并发模式失败

G1优点：可预测的停顿模型，G1的并行和并发的特性更好的适合利用多核CPU；从整体来看G1使用的是标记整理算法，从局部上看使用的是标记清除算法；

G1收集器与之前的收集器相比，G1将堆内存划分为多个大小固定的的小区域，每个region在运行时可以被指定为eden,survivor,old区域，这样每次进行回收的时候都不是回收整个区域，而是回收一块区域就可以了；

**CSet是G1实现可预测停顿的关键数据结构**。它代表了**一次垃圾回收过程中，被选择出来等待回收的Region的集合**。G1的每次回收都是针对一个CSet进行的，而不是整个代。转化为在一个有限时间内对垃圾的回收；

### 12.synchronized锁升级的流程，说一下什么时候会发生锁升级？我三阶段答上来了，但是升级的触发时机有点忘了

答：锁有四种状态：无锁，偏向锁，轻量级锁，重量级锁。

无锁当有一个线程访问同步块时，JVM会启用偏向锁，当其他线程尝试获取偏向锁的时候偏向锁会升级为轻量级锁，轻量级锁自旋失败比如超过最大尝试此说，轻量级锁会升级为重量级锁。

### 15.事务传播机制，如何设置传播机制

答：事务的传播机制主要有七种：REQUIRED：如果存在当前事务就加入该事务，否则创建一个新事务

​							  SUPPORTS：如果存在当前事务就加入，否则以非事务状态下运行。

​							 MANDATORY：必须在一个已有的事务中运行，否则抛出异常。

​							REQUIRES_NEW：无论存在不存在都会创建一个新事务，挂起当前事务

​							NOT_SUPPORTED：以非事务方式运行，如果存在当前事务。则挂起它

​							NEVER：必须在非事务环境下运行，否则抛出异常

​							NESTED：如果存在当前事务，则在嵌套中运行，否则创建一个新事务

​	1.在方法上使用@Transactional注解指定传播行为

​	2.在Spring XML文件中定义事务的传播规则

​	3.通过编码动态控制传播行为

### 36.什么是IOC，AOP

答：IOC是控制反转，它是一种思想而不是技术，描述的是java开发领域对象的创建和管理问题，核心思想就是双方互不依赖，由第三方容器管理资源。

​	AOP是面向切面编程，目的是将横切关注点（日志记录，权限控制，事务管理，接口限流等）从核心业务中剥离出来，通过动态代理、字节码操作等技术，实现代码的复用和解耦，提高代码的可维护性和可扩展性。常见的几个点：

​	目标：被通知对象； 代理：向目标对象应用通知之后的代理对象；连接点：目标所属类中所有方法都是连接点；

​	切入点：被切面、拦截的连接点； 通知：增强的逻辑、代码； 切面：切入点+通知

### 132.为什么使用跳表而不使用平衡树

​	跳表结构体：对象的元素值，元素权重值，后向指针，节点的level数组保存每层上的前向指针与跨度；

### 296.countdownlatch的底层原理是什么

答：`CountDownLatch` 是共享锁的一种实现,它默认构造 AQS 的 `state` 值为 `count`。当线程使用 `countDown()` 方法时,其实使用了`tryReleaseShared`方法以 CAS 的操作来减少 `state`,直至 `state` 为 0 。当调用 `await()` 方法的时候，如果 `state` 不为 0，那就证明任务还没有执行完毕，`await()` 方法就会一直阻塞，也就是说 `await()` 方法之后的语句不会被执行。直到`count` 个线程调用了`countDown()`使 state 值被减为 0，或者调用`await()`的线程被中断，该线程才会从阻塞中被唤醒，`await()` 方法之后的语句得到执行。

### 331.为什么二级缓存解决不了AOP代理场景下的循环依赖

答：代理：实例化-》属性注入-》初始化-》创建代理；

**A 开始创建：** Spring 尝试获取 A，缓存里没有。于是实例化 A（通过反射调构造函数），此时 A 是个**半成品**。

**存入三级缓存：** Spring 把 A 的“配方”（ObjectFactory）存入**三级缓存**。

**A 填充属性：** 发现需要 B，去缓存里找 B，没找到，于是开始**创建 B**。

**B 开始创建：** 实例化 B，把 B 的“配方”存入**三级缓存**。

**B 填充属性：** 发现需要 A，去缓存里找：

- 一级没有，二级没有，**三级找到了 A 的配方！**
- 通过配方拿到 A 的引用（如果是 AOP 则是代理对象），把 A 放入**二级缓存**，并从三级缓存删除 A。

**B 完成：** B 拿到了 A 的引用，成功完成初始化，进入**一级缓存**。

**A 完成：** A 拿到了 B 的引用，继续完成初始化，进入**一级缓存**。

其实我们可以理解为三级缓存就是一个工厂，用于生成对象；Spring有一个设计原则位尽量再Bean初始化完成之后再创建代理；如果没有三级缓存，这不就卡在这了吗，但是如果是有三级缓存，三级缓存就是创建代理的配方，这样就可以创建代理对象了啊；

### 336.接口和抽象类的区别

答：**设计目的**：接口主要用于对类的行为进行约束，你实现了某个接口就具有了对应的行为。抽象类主要用于代码复用，强调的是所属关系。

**继承和实现**：一个类只能继承一个类（包括抽象类），因为 Java 不支持多继承。但一个类可以实现多个接口，一个接口也可以继承多个其他接口。

**成员变量**：接口中的成员变量只能是 `public static final` 类型的，不能被修改且必须有初始值。抽象类的成员变量可以有任何修饰符（`private`, `protected`, `public`），可以在子类中被重新定义或赋值。

### 339.内存泄露跟OOM

### 343.AOP实现的底层逻辑

答：基本原理：动态代理

​	JDK动态代理：1.目标类必须实现了这个接口

​				原理：利用反射机制，在内存中创建了一个实现了相同接口的代理类，当你调用接口方法时，实际上调用的时代理类的方法，由代理类去执行增强逻辑和原业务逻辑

​	CGLIB代理：1.目标类没有实现接口

​				原理：通过底层字节码技术，为目标类生成一个子类，代理类通过重写父类的方法来实现增强；

​				局限：因为是因为继承，所以不能代理final修饰的类或方法；
