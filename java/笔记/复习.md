​	每次反射访问方法或字段时，JVM必须执行安全检查。

​	破坏了JIT编译的优化，反射时黑盒，由于反射时运行时的动态行为，JIT很难预测下一步会发生什么，因此无法堆反射代码进性内联优化。

### 20.HashSet,LinkedHashSet,TreeSet

答：1.HashSet,LinkedHashSet,TreeSet都是Set接口的实现类，都能保证元素唯一，并且都不是线程安全；

​	2.HashSet,LinkedHashSet和TreeSet的主要区别在于底层数据结构不同，HashSet的底层数据结构是哈希表（基于HashMap实现）。LinkedHashSet的底层数据结构是链表和哈希表，元素的插入和取出顺序满足FIFO。TreeSet的底层数据结构是红黑树，元素是有序的，排序的方式有自然排序和定制排序。

### 23.LinkedHashMap 和HashMap有什么区别

答：`LinkedHashMap` 和 `HashMap` 都是 Java 集合框架中的 Map 接口的实现类。它们的最大区别在于迭代元素的顺序。`HashMap` 迭代元素的顺序是不确定的，而 `LinkedHashMap` 提供了按照插入顺序或访问顺序迭代元素的功能。此外，`LinkedHashMap` 内部维护了一个双向链表，用于记录元素的插入顺序或访问顺序，而 `HashMap` 则没有这个链表。因此，`LinkedHashMap` 的插入性能可能会比 `HashMap` 略低，但它提供了更多的功能并且迭代效率相较于 `HashMap` 更加高效。

### 29.线程异常还是复用

答：**使用`execute()`提交任务**：当任务通过`execute()`提交到线程池并在执行过程中抛出异常时，如果这个异常没有在任务内被捕获，那么该异常会导致当前线程终止，并且异常会被打印到控制台或日志文件中。线程池会检测到这种线程终止，并创建一个新线程来替换它，从而保持配置的线程数不变。

**使用`submit()`提交任务**：对于通过`submit()`提交的任务，如果在任务执行中发生异常，这个异常不会直接打印出来。相反，异常会被封装在由`submit()`返回的`Future`对象中。当调用`Future.get()`方法时，可以捕获到一个`ExecutionException`。在这种情况下，线程不会因为异常而终止，它会继续存在于线程池中，准备执行后续的任务。

### 46.事务传播机制，如何设置传播机制

答：事务的传播机制主要有七种：REQUIRED：如果存在当前事务就加入该事务，否则创建一个新事务

​							  SUPPORTS：如果存在当前事务就加入，否则以非事务状态下运行。

​							 MANDATORY：必须在一个已有的事务中运行，否则抛出异常。

​							REQUIRES_NEW：无论存在不存在都会创建一个新事务，挂起当前事务

​							NOT_SUPPORTED：以非事务方式运行，如果存在当前事务。则挂起它

​							NEVER：必须在非事务环境下运行，否则抛出异常

​							NESTED：如果存在当前事务，则在嵌套中运行，否则创建一个新事务

​	1.在方法上使用@Transactional注解指定传播行为

​	2.在Spring XML文件中定义事务的传播规则

​	3.通过编码动态控制传播行为

### 65.静态代理和动态代理有什么区别

答：静态代理就是我们队目标对象的每个方法的增强都是手动完成的。非常不灵活（比如接口一旦新增加方法，目标对象和代理对象都要进行修改），需要对每个目标类单独写一个代理类；

静态代理步骤：定义一个接口及其实现类，创建一个代理类同样实现这个接口，将目标对象注入进代理类，然后在代理类对应方法调用目标类的对应方法。

动态代理实现更加灵活，不用必须实现接口，可以直接代理实现类。并且不需要针对每个目标类都创建一个代理类，另外静态代理中接口一旦新增加方法，目标对象和代理都需要进行修改；

代理模式主要的功能就是扩展目标对象的功能；比如在目标对象的某个方法执行前后你可以增加一些自定义的操作；

### 66.常用线程池类型

答：fixedthredapool（固定线程池） 核心线程数==最大线程数；队列为无界队列，不会创建额外线程；  适用场景，并发量固定  优点：控制线程数量，避免线程重复创建与销毁；任务可以排队等待执行；  缺点：队列无限大，可能导致oom;

​     cachedThreadPool(可缓存线程池)  核心线程数为0，最大线程数为Integer.MAX_VALUE; 空闲线程60秒自动回收；适用场景：任务量波动大，执行时间段；优点：高并发时可以快速创建线程，空闲线程被回收，减少资源浪费； 缺点：频繁创建线程与销毁，容易导致oom；

​	singleThreadPool(单一线程池) 只有一个线程执行任务，执行按队列顺序执行；适用场景：必须顺序执行；优点：顺序执行，线程安全，不需要额外同步；缺点：并发能力低，如果任务长时间阻塞，会阻塞后续任务；

### 71.为什么使用跳表不使用平衡树

​	归并排序：就是分开几个组，然后组再排             稳定    nlogn

​	快速排序：分为两个部分，左边都是小于的，右边都是大于的   不稳定  nlogn

​	堆排序：以二叉树的形式实现，通常有最小堆，最大堆；堆是一个完全二叉树，父节点大于子节点；
